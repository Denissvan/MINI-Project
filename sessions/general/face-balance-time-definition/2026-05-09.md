# Face Balance Time Definition

- Workshop identifier: general / not specified
- Demand track: face-balance-time-definition
- Date: 2026-05-09
- Scope: analyze how the face balance page defines and calculates test time.

## Current Finding

- The face balance page is implemented in `UI/FormView/FrCount`.
- Real-time displayed light-box times use `PT_SET.lefttime`, `PT_SET.otptime`, `PT_SET.righttime`, and `PT_SET.udtime`.
- Average face times and balance rate use database query results written into `PT_SET.lefttimedb`, `PT_SET.otptimedb`, `PT_SET.righttimedb`, and `PT_SET.ratedb`.
- Each test-time record is written from `WS.RunTest` using `Swtime.ElapsedMilliseconds / 1000`.
- `Swtime` starts when the station test thread starts and stops in `finally`, so it includes the station test thread duration, not only PC-side image/test algorithm time.
- Database rows are stored in `product/TestimeDataBase/<yyyy_MM>.db`, table `TyyyyMMdd`, with fields `TIME`, `WSID`, `LBID`, and `TESTTIME`.

## Formula

- Per-face average:
  - left average = sum(TESTTIME where LBID = 1) / count(LBID = 1)
  - right average = sum(TESTTIME where LBID = 2) / count(LBID = 2)
  - OTP average = sum(TESTTIME where LBID = 3) / count(LBID = 3)
- Face balance rate:
  - `(left average + OTP average + right average) / (max(left average, OTP average, right average) * 3)`

## Notes

- Existing code comments label `LBID = 2` and `LBID = 3` comments inconsistently, but the write-side mapping is left = 1, right = 2, OTP = 3.
- Times are integer seconds because elapsed milliseconds are divided by integer `1000` before being assigned to `double`.
